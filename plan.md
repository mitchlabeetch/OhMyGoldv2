# OhMyGold — Master Project Plan

## Overview
Build the complete project foundation for OhMyGold, a full-stack gym management & client solution system for Gold's Gym France. This is the **pre-development documentation & planning phase** that produces all design systems, feature specifications, asset libraries, and implementation roadmaps required for full development.

## Deliverables
A complete, self-contained project directory at `/mnt/agents/output/ohmygold/` containing:
- Gold's Gym Resource Base (Team 1)
- Resamania Analysis & Feature Matrix (Team 2)
- Technical Architecture & Implementation Roadmap (Team 3)
- QA-Validated Final Output (Team 4)

## Teams & Stages

### Stage 1 — Parallel Research & Analysis (Teams 1 & 2)
- **Team 1 (Gold's Gym Resource Base)**: Analyze all uploaded HTML pages and image assets, extract brand philosophy, offers, locations, wording. Create comprehensive documentation, rename/reorganize all assets, build DESIGN.MD, and create a steering document.
- **Team 2 (Resamania Analysis & Feature List)**: Analyze Resamania HTML pages for every feature, conduct web research on gym management best practices, build detailed feature lists with workflows, create role-permission matrices (6 roles), and mobile accessibility mapping.

**Dependencies**: None. Both teams work in parallel using the uploaded assets.

### Stage 2 — Technical Architecture & Roadmap (Team 3)
- Read outputs from Teams 1 & 2.
- Build complete technical architecture (webapp + mobile), tech stack, design systems, asset references.
- Create phased, itemized implementation roadmap with LLM-evolutive documentation (tickboxes, notes, agent-optimized prompts).
- Output must cover every run until final product launch.

**Dependencies**: Stage 1 outputs (Teams 1 & 2).

### Stage 3 — Full QA & Refinement (Team 4)
- Re-read original user requirements word-by-word.
- Audit all outputs from Teams 1, 2, and 3 with severity.
- Apply corrections, improvements, polishes.
- Validate 100% completeness before declaring ready.

**Dependencies**: Stage 2 output (Team 3).

### Stage 4 — Integration & Final Assembly
- Orchestrator merges all team outputs into the final `ohmygold/` directory.
- Ensure all cross-references are valid.
- Deliver final summary.

## Skills Used
- `deep-research-swarm`: For competitive analysis and best practices research (Team 2 web research, Team 3 tech research).
- `report-writing`: For structured documentation output from all teams.
- `vibecoding-webapp-swarm`: Will be loaded by Team 3 for webapp tech stack guidance.
- `vibecoding-general-swarm`: Will be loaded by Team 3 for React Native mobile app guidance.

## Notes
- All outputs are Markdown files for LLM-friendly evolution.
- Asset management includes multiple sizes/formats.
- Every feature must have concrete workflow descriptions and success criteria.
- Roadmap phases must be dev-friendly with agent-optimized launch prompts.
